# ТЗ: подпроект mwps_provider_client

**Author:** Vasiliy Zdanovskiy  
**email:** vasilyvz@gmail.com

Клиент провайдера Model Workplace Server: реализует контракт из корневой документации [provider_client_standard](../../docs/standards/provider_client_standard.md). Для рабочего места модели **сервер Model Workplace Server — просто отдельный провайдер**; данный пакет — клиент, обращающийся к mwps-adapter (или к Model Workplace Server напрямую). Общий контекст: [SPEC](../../docs/plans/refactoring_adapter_structure/SPEC.md).

---

## 1. Назначение

- **mwps_provider_client** — пакет «клиент провайдера Model Workplace Server»: единый API (chat, embed, healthcheck, ошибки) для model_workspace.
- Рабочее место модели использует только этот контракт; для него Model Workplace Server — один из провайдеров наравне с другими (OpenAI, Anthropic и т.д.).
- Транспорт: WebSocket к mwps-adapter (или HTTP к Model Workplace Server) — внутренняя деталь клиента.

---

## 2. Контракт

- Реализация **обязана** следовать [docs/standards/provider_client_standard.md](../../docs/standards/provider_client_standard.md): validate_config(), healthcheck(), chat(), embed(), supports_tools(), normalize_response(), map_error(); флаги возможностей; категории ошибок.
- Конфигурация — по [provider_client_config_standard](../../docs/standards/provider_client_config_standard.md) (секция провайдера `mwps`).

---

## 3. Зависимости и база

- Клиент **строится на базе клиента адаптера** (mcp-proxy-adapter: WebSocket к серверу mwps-adapter). Специфика и формат работы с Model Workplace Server/адаптером скрыты внутри пакета.
- Зависимости: клиент адаптера (mcp-proxy-adapter) и при необходимости httpx для fallback или прямого HTTP к Model Workplace Server.

---

## 4. Контейнер

- Клиент — библиотека; отдельного контейнера нет. Запускается в составе model-workspace-server.
