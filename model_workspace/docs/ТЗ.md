# ТЗ: подпроект model_workspace

**Author:** Vasiliy Zdanovskiy  
**email:** vasilyvz@gmail.com

Техническое задание на подпроект «рабочее место модели»: приложение, использующее **клиенты провайдеров** (mwps_provider_client, redis_provider_client). Общий контекст — в корне репозитория: `docs/plans/refactoring_adapter_structure/SPEC.md`.

---

## 1. Назначение

- **model-workspace-server** — приложение «рабочее место модели»: оркестрация чата с моделью, сессии, контекст, вызов инструментов через MCP Proxy.
- **Внутри контейнера/процесса нет Redis и Model Workplace Server.** Все обращения к хранилищу и к модели — через **клиенты провайдеров**: **mwps_provider_client** (модель) и **redis_provider_client** (хранилище).
- **Для рабочего места модели сервер Model Workplace Server — просто отдельный провайдер** наравне с другими (OpenAI, Anthropic и т.д.); единый контракт — root docs/standards/provider_client_standard.

---

## 2. Зависимости от подпроектов

- **mwps_provider_client** (пакет `mwps-provider-client` / `mwps_provider_client`) — клиент провайдера Model Workplace Server: chat, embed, healthcheck; контракт по provider_client_standard. Подключение к mwps-adapter (или к Model Workplace Server) — внутренняя деталь клиента.
- **redis_provider_client** (пакет `redis-provider-client` / `redis_provider_client`) — клиент к redis-adapter: API хранилища (execute + обёртки get, set, hgetall, …).
- Подключение к серверам адаптеров — по WebSocket через прокси; конфигурация (URL прокси, идентификаторы серверов) — в конфиге подпроекта или в конфиге провайдеров.

---

## 3. Конфигурация

- Собственные конфиги в каталоге подпроекта (`config/`): URL прокси, параметры провайдеров (provider_clients по docs/standards/provider_client_config_standard), модель по умолчанию, лимиты, политики команд и т.д.
- Без хардкода адресов и учётных данных; при необходимости — ссылка на общие сертификаты (mtls_certificates в корне репозитория).

---

## 4. Контейнер

- Контейнер **model-workspace-server**: только приложение данного подпроекта + зависимости (в т.ч. mwps_provider_client и redis_provider_client). Ни Redis, ни Model Workplace Server внутри образа не запускаются.

---

## 5. Функциональность (обзор)

- Инициализация сессии, чат с моделью (через **mwps_provider_client** — Model Workplace Server как один из провайдеров), сохранение/загрузка сообщений (через **redis_provider_client** при необходимости).
- Вызов инструментов через MCP Proxy (list_servers, call_server, help и т.д.), встраивание ответов в контекст модели.
- Регистрация/интеграция с прокси по требованиям общего ТЗ (если приложение само регистрируется как сервис).

Детализация сценариев и API — в отдельной спецификации или в общем SPEC в корне.
