# ТЗ: подпроект ollama_provider_client

**Author:** Vasiliy Zdanovskiy  
**email:** vasilyvz@gmail.com

Клиент провайдера Ollama: реализует контракт из корневой документации [provider_client_standard](../../docs/standards/provider_client_standard.md). Для рабочего места модели **сервер Ollama — просто отдельный провайдер**; данный пакет — клиент, обращающийся к ollama-adapter (или к Ollama напрямую). Общий контекст: [SPEC](../../docs/plans/refactoring_adapter_structure/SPEC.md).

---

## 1. Назначение

- **ollama_provider_client** — пакет «клиент провайдера Ollama»: единый API (chat, embed, healthcheck, ошибки) для model_workspace.
- Рабочее место модели использует только этот контракт; для него Ollama — один из провайдеров наравне с другими (OpenAI, Anthropic и т.д.).
- Транспорт: WebSocket к ollama-adapter (или HTTP к Ollama) — внутренняя деталь клиента.

---

## 2. Контракт

- Реализация **обязана** следовать [docs/standards/provider_client_standard.md](../../docs/standards/provider_client_standard.md): validate_config(), healthcheck(), chat(), embed(), supports_tools(), normalize_response(), map_error(); флаги возможностей; категории ошибок.
- Конфигурация — по [provider_client_config_standard](../../docs/standards/provider_client_config_standard.md) (секция провайдера `ollama`).

---

## 3. Зависимости

- Минимум: httpx (или транспорт для WebSocket к ollama-adapter). Без mcp-proxy-adapter в рантайме, если клиент сам реализует протокол к адаптеру.

---

## 4. Контейнер

- Клиент — библиотека; отдельного контейнера нет. Запускается в составе model-workspace-server.
