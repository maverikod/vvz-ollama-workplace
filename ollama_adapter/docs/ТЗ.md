# ТЗ: подпроект ollama_adapter

**Author:** Vasiliy Zdanovskiy  
**email:** vasilyvz@gmail.com

Техническое задание на подпроект Ollama-адаптера: клиент и сервер на базе mcp-proxy-adapter. Общий контекст — в корне репозитория: `docs/plans/refactoring_adapter_structure/SPEC.md`.

---

## 1. Назначение

- **Сервер ollama-adapter:** Ollama + сервер API на базе mcp-proxy-adapter; регистрация на MCP Proxy; доступ к Ollama только через адаптер (WebSocket).
- **Клиент Ollama на базе адаптера:** передаёт запросы серверу по WebSocket, воспроизводит полный API Ollama (тоннель); вызывающий код получает питоновские объекты.

---

## 2. Транспорт

- Взаимодействие клиент ↔ сервер — **только WebSocket** (через прокси).

---

## 3. Классы протокола

- **Отдельный класс для запроса/ответа Ollama** — структура: имя метода/эндпоинта, параметры (model, messages, prompt и т.д.), тело ответа (например, JSON: `method`, `params`, `result` / `error`).
- **Отдельный класс для ошибок Ollama** — тип для ошибок (HTTP 4xx/5xx, протокол, таймаут); на клиенте — исключение этого класса.

---

## 4. Клиент: один метод + обёртки

- **Базовый метод:** `execute(method_name: str, **params)` — отправляет на сервер имя метода и параметры по WebSocket, получает ответ, десериализует и возвращает питоновские объекты (dict, list, str и т.д.).
- **Остальные методы** — обёртки над `execute()` (tags, chat, generate, embeddings, pull, push, delete, show, copy, create, ps, version). **Валидация параметров** выполняется **в обёртках**, не в `execute()`.

**Полный набор методов (эндпоинты Ollama):**

| Метод клиента       | Назначение                    |
|---------------------|-------------------------------|
| `tags()` / `list_models()` | Список моделей (GET /api/tags) |
| `chat(model, messages, …)` | Чат (POST /api/chat)          |
| `generate(model, prompt, …)` | Генерация (POST /api/generate) |
| `embeddings(model, input)` | Эмбеддинги (POST /api/embeddings) |
| `pull(model)`       | Скачивание модели             |
| `push(model)`       | Выгрузка модели               |
| `delete(model)`     | Удаление модели               |
| `show(model)`       | Информация о модели           |
| `copy(source, dest)`| Копирование модели            |
| `create(name, modelfile, …)` | Создание из Modelfile    |
| `ps()` / `running()`| Запущенные модели             |
| `version()`         | Версия Ollama                 |

---

## 5. Сервер ollama-adapter

- Принимает запрос, **десериализует** (имя метода + параметры).
- Вызывает **нативный HTTP-клиент** к локальному Ollama (соответствующий эндпоинт: /api/tags, /api/chat и т.д.).
- Ответ Ollama **сериализует в JSON** и отправляет клиенту по WebSocket.

---

## 6. Контейнер

- Контейнер **ollama-adapter**: Ollama + сервер mcp-proxy-adapter; образ собирается из данного подпроекта.
