# ТЗ: подпроект redis_adapter

**Author:** Vasiliy Zdanovskiy  
**email:** vasilyvz@gmail.com

Техническое задание на подпроект Redis-адаптера: клиент и сервер на базе mcp-proxy-adapter. Общий контекст — в корне репозитория: `docs/plans/refactoring_adapter_structure/SPEC.md`.

---

## 1. Назначение

- **Сервер redis-adapter:** Redis + сервер API на базе mcp-proxy-adapter; регистрация на MCP Proxy; доступ к Redis только через адаптер (WebSocket).
- **Клиент Redis на базе адаптера:** передаёт команды серверу по WebSocket, воспроизводит API Redis (тоннель); вызывающий код получает питоновские объекты.

---

## 2. Транспорт

- Взаимодействие клиент ↔ сервер — **только WebSocket** (через прокси).

---

## 3. Классы протокола

- **Отдельный класс для запроса/ответа Redis** — структура: имя команды, параметры, тело ответа (например, JSON: `command`, `params`, `result` / `error`).
- **Отдельный класс для ошибок Redis** — тип для ошибок (ответ Redis, исключение нативного клиента, ошибка протокола); на клиенте — исключение этого класса.

---

## 4. Клиент: один метод + обёртки

- **Базовый метод:** `execute(command_name: str, *args, **kwargs)` — отправляет на сервер имя команды и параметры по WebSocket, получает ответ, десериализует и возвращает питоновские объекты (str, int, list, dict, bytes и т.д.).
- **Остальные методы** — обёртки над `execute()` (get, set, hgetall, lrange, …). **Валидация параметров** (наличие обязательных, отсутствие лишних, типы) выполняется **в обёртках**, не в `execute()`.
- Эталон набора команд: [Redis Commands](https://redis.io/docs/latest/commands/), [redis-py Commands](https://redis.readthedocs.io/en/stable/commands.html) (String, Hash, List, Set, Sorted Set, Bitmap, Pub/Sub, транзакции, сервер и т.д.).

---

## 5. Сервер redis-adapter

- Принимает запрос, **десериализует** (имя команды + параметры).
- Вызывает **нативный клиент Redis** (redis-py) с этой командой и параметрами.
- Ответ Redis **сериализует в JSON** и отправляет клиенту по WebSocket.

---

## 6. Контейнер

- Контейнер **redis-adapter**: Redis + сервер mcp-proxy-adapter; образ собирается из данного подпроекта (или из корня монорепо с указанием подпроекта).
