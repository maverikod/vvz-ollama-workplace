# ТЗ: подпроект redis_provider_client

**Author:** Vasiliy Zdanovskiy  
**email:** vasilyvz@gmail.com

Клиент провайдера Redis (хранилище): обращается к redis-adapter по WebSocket; для model_workspace предоставляет API хранилища (get, set, hgetall, lrange и т.д.). Общий контекст: [SPEC](../../docs/plans/refactoring_adapter_structure/SPEC.md).

---

## 1. Назначение

- **redis_provider_client** — пакет «клиент к redis-adapter»: один базовый метод execute(command, *args) и обёртки (get, set, …); валидация в обёртках.
- Рабочее место модели использует этот клиент для сессий, сообщений, кэша и т.д.; внутри модели нет Redis.
- Транспорт: WebSocket к redis-adapter (через прокси) — внутренняя деталь.

---

## 2. Контракт

- API в стиле redis-py: execute(command_name, *args, **kwargs) + обёртки. Классы запроса/ответа и ошибок Redis — по SPEC §7.
- Конфигурация: URL прокси, идентификатор сервера redis-adapter, сертификаты.

---

## 3. Контейнер

- Клиент — библиотека; отдельного контейнера нет. Запускается в составе model-workspace-server.
